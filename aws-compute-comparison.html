offset 004 — rule of thumb

For a new service in 2026

Default choice

ECS on Fargate. Simplest operational path for most new AWS-only services.

Drop to EC2 launch type

Only when cost at scale or GPU/specialized hardware demands it.

Reach for EKS

Only with a real Kubernetes-specific reason — team expertise, multi-cloud, CRDs.

Use Lambda

For the event-driven slices — notification sends, bulk-update triggers, Kafka-adjacent glue work — not the whole app.
