01

Built-in Functions

The functions always available in Python — no import required. These are the verbs you reach for constantly: measuring, converting, ordering, and inspecting data.

no importglobal scopereusable

len() sizing

Returns the number of items in a sequence or collection.

# works on str, list, tuple, dict, set
len("python")   ## 6
len([1, 2, 3])   ## 3

type() / isinstance() inspection

Check an object's exact type, or test membership against one or more types.

type(5)                    ## <class 'int'>
isinstance(5, int)         ## True
isinstance("x", (int, str)) ## True

print() output

Writes text to the console. Accepts multiple values, a custom separator, and line ending.

print("a", "b", sep="-", end="!\n")
## a-b!

range() sequence

Generates a lazy, immutable sequence of numbers — the standard tool for counted loops.

list(range(5))        ## [0,1,2,3,4]
list(range(2, 10, 2)) ## [2,4,6,8]

sorted() ordering

Returns a new sorted list from any iterable. Original is untouched. Accepts key= and reverse=.

sorted([3, 1, 2])                 ## [1,2,3]
sorted(words, key=len, reverse=True)

reversed() ordering

Returns a reverse iterator over a sequence. Wrap in list() to see it.

list(reversed([1, 2, 3])) ## [3,2,1]

enumerate() iteration

Pairs each item in an iterable with its index. The go-to when you need position and value.

for i, val in enumerate(["a","b"]):
    print(i, val)
## 0 a
## 1 b

zip() iteration

Combines two or more iterables element-wise into tuples. Stops at the shortest input.

list(zip([1,2], ["a","b"]))
## [(1,'a'), (2,'b')]

map() / filter() transformation

Apply a function across an iterable (map), or keep only items where it's truthy (filter).

list(map(str.upper, ["a","b"]))
list(filter(lambda x: x>0, [-1,2,3]))

sum() / min() / max() aggregation

Numeric aggregates over any iterable. min/max accept key= for custom comparisons.

sum([1,2,3])          ## 6
max(words, key=len)   ## longest word

abs() / round() math

Absolute value, and rounding to N decimal places (banker's rounding on .5 ties).

abs(-7)        ## 7
round(3.14159, 2) ## 3.14

all() / any() boolean logic

all() is True if every item is truthy; any() is True if at least one is.

all([1, 1, 0])  ## False
any([0, 0, 1])  ## True

int() / float() / str() / bool() casting

Convert between core types. list(), tuple(), dict(), set() convert between collections.

int("42")     ## 42
str(3.14)     ## '3.14'
list("abc")   ## ['a','b','c']

input() i/o

Pauses execution and reads a line of text from the user as a string.

name = input("Your name: ")

open() i/o

Opens a file. Always pair with with so the file closes automatically.

with open("data.txt") as f:
    text = f.read()

id() / dir() / help() introspection

Object identity, listing an object's attributes, and pulling up interactive documentation.

dir(str)     ## every str method
help(str.split)

format() / f-strings formatting

Format values into strings. F-strings (Python 3.6+) are the modern, preferred approach.

name, age = "Ana", 30
f"{name} is {age} years old"
f"{3.14159:.2f}"  ## '3.14'
02

Strings

Immutable sequences of characters. Every method below returns a new string — the original is never changed in place.

immutableorderedindexed
Creating & accessing

Literals create

Single, double, or triple quotes (for multi-line text).

s = 'hello'
s = "hello"
s = """multi
line"""

Indexing & slicing access

Zero-based indexing; negative indices count from the end. Slices are [start:stop:step].

s = "python"
s[0]      ## 'p'
s[-1]     ## 'n'
s[1:4]    ## 'yth'
s[::-1]   ## 'nohtyp' (reversed)

Concatenation & repetition combine

+ joins strings; * repeats them.

"py" + "thon"   ## 'python'
"ab" * 3       ## 'ababab'

Escape sequences syntax

Special characters embedded with a backslash.

"\n" newline   "\t" tab
"\\" backslash "\'" quote
Methods — inspecting

.upper() .lower() .title() case

Change letter casing without touching the original string.

"Py".upper()      ## 'PY'
"Py".lower()      ## 'py'
"py rocks".title() ## 'Py Rocks'

.strip() .lstrip() .rstrip() clean

Remove whitespace (or given characters) from the ends.

"  hi  ".strip()   ## 'hi'
"xxhixx".strip("x") ## 'hi'

.find() .index() .count() search

find returns -1 if missing; index raises an error; count tallies occurrences.

"hello".find("l")   ## 2
"hello".count("l")  ## 2

.startswith() .endswith() check

Boolean checks on prefixes and suffixes.

"report.pdf".endswith(".pdf") ## True

.isdigit() .isalpha() .isspace() validate

Test the character composition of a string.

"123".isdigit()   ## True
"abc".isalpha()   ## True

in membership

Check whether a substring exists within a string.

"the" in "gather"  ## True
Methods — transforming

.split() .rsplit() break apart

Splits a string into a list, on whitespace by default or a given delimiter.

"a,b,c".split(",")   ## ['a','b','c']
"one two".split()    ## ['one','two']

.join() assemble

The reverse of split — glues a list of strings together with a separator.

"-".join(["a","b","c"])
## 'a-b-c'

.replace() substitute

Returns a copy with all occurrences of a substring swapped out.

"cat hat".replace("at", "og")
## 'cog hog'

.ljust() .rjust() .zfill() pad

Pad a string to a fixed width — handy for aligned tables or numeric IDs.

"7".zfill(3)      ## '007'
"hi".rjust(5, ".") ## '...hi'

f-strings & .format() build

Insert values into a template string with inline expressions.

n = 7
f"squared: {n**2}"  ## 'squared: 49'
"{}-{}".format("a","b")

Slicing tricks idiom

Reverse a string, or grab every Nth character, with slice syntax.

s[::-1]     ## reverse
s[::2]      ## every other char
03

Lists

Ordered, mutable collections. Methods below modify the list in place and usually return None — don't write x = x.append(1).

mutableorderedindexedallows duplicates

Creating a list create

Square brackets, mixed types allowed, can be nested.

nums = [1, 2, 3]
mixed = [1, "two", 3.0, [4,5]]

Indexing & slicing access

Same rules as strings — zero-based, negative indices, slices return a new list.

nums[0]     ## first item
nums[-1]    ## last item
nums[1:3]   ## sub-list

.append() add

Adds a single item to the end of the list.

nums.append(4)   ## [1,2,3,4]

.extend() add

Appends every item from another iterable — unlike append, it doesn't nest.

nums.extend([4,5]) ## [1,2,3,4,5]

.insert() add

Inserts an item at a specific index, shifting later items right.

nums.insert(1, "x")  ## [1,'x',2,3]

.remove() .pop() .clear() remove

remove deletes by value; pop deletes by index and returns it; clear empties the list.

nums.remove(2)   ## deletes value 2
nums.pop()       ## removes & returns last
nums.pop(0)      ## removes & returns index 0

.sort() order

Sorts the list in place. Use sorted() instead if you need the original preserved.

nums.sort()
nums.sort(reverse=True)
words.sort(key=len)

.reverse() order

Reverses the list in place.

nums.reverse()

.copy() duplicate

Shallow copy — needed because b = a just creates a second name for the same list.

b = a.copy()
b = a[:]         ## equivalent

.index() .count() search

Find the first position of a value, or count how many times it appears.

nums.index(3)   ## position of first 3
nums.count(3)   ## how many 3's

List comprehensions idiom

The Pythonic way to build a list from a transformation, optionally filtered.

[x*x for x in range(5)]
## [0,1,4,9,16]
[x for x in nums if x % 2 == 0]

+ / * / in operators

Concatenate, repeat, and test membership — same as strings.

[1,2] + [3,4]  ## [1,2,3,4]
[0] * 3        ## [0,0,0]
3 in nums      ## True
04

Tuples

Ordered, immutable collections. Once created, a tuple can't be changed — which makes it fast, hashable, and safe to use as a dictionary key.

immutableorderedhashable

Creating a tuple create

Parentheses (often optional). A single-item tuple needs a trailing comma.

point = (3, 4)
single = (5,)     ## comma required!
no_parens = 1, 2, 3

Indexing & slicing access

Read exactly like a list — you just can't assign to an index.

point[0]     ## 3
point[1:]    ## (4,)

Unpacking idiom

Assign tuple items directly to variables in one line — the most common tuple use case.

x, y = point
name, *rest = ("Ana", 30, "NY")

Immutability behavior

Attempting to modify a tuple raises a TypeError. Rebuild a new tuple instead.

point[0] = 9
## TypeError: 'tuple' object
## does not support item assignment

.count() .index() search

The only two methods tuples have — everything else is read-only behavior.

t = (1, 2, 2, 3)
t.count(2)   ## 2
t.index(3)   ## 3

Why choose a tuple? reference

Fixed records (coordinates, RGB values), dictionary keys, and function returns of multiple values.

def minmax(nums):
    return min(nums), max(nums)
lo, hi = minmax([3,1,9])
05

Dictionaries

Mutable key → value mappings. Keys must be unique and hashable (strings, numbers, tuples); insertion order is preserved since Python 3.7.

mutablekey-valueunique keys

Creating a dict create

Curly braces with key: value pairs.

user = {"name": "Ana", "age": 30}

Access & .get() access

Bracket access raises KeyError on a missing key; .get() returns None (or a default) instead.

user["name"]           ## 'Ana'
user.get("email")       ## None
user.get("email", "n/a")

Setting & .update() write

Assign a new or existing key; update() merges in another dict at once.

user["age"] = 31
user.update({"city": "NY"})

.pop() .popitem() del remove

Remove by key (returning its value), remove the last-inserted pair, or delete outright.

user.pop("age")     ## returns value, removes key
del user["city"]

.keys() .values() .items() iterate

Views onto the dict's keys, values, or (key, value) pairs — the standard iteration trio.

for k, v in user.items():
    print(k, v)

.setdefault() idiom

Gets a key's value, inserting a default first if it's missing. Great for building groups/counts.

groups = {}
groups.setdefault("a", []).append(1)

in membership

Checks against keys by default — fast, O(1) lookup.

"name" in user   ## True

Dict comprehensions idiom

Build a dictionary from an iterable in a single expression.

{x: x**2 for x in range(5)}
## {0:0, 1:1, 2:4, 3:9, 4:16}

Nested dicts structure

Values can be any type, including other dicts — how JSON-like data is modeled in Python.

data = {"user": {"name": "Ana", "roles": ["admin"]}}
data["user"]["roles"]
