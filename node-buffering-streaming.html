Buffering and Streaming in Node.js

Buffering stores raw bytes in memory, while streaming moves data in chunks so large operations can happen without blocking the app.

1. What a Buffer is

A Buffer holds binary data in memory. It is useful when working with files, sockets, and network traffic.

const buf = Buffer.from('Hello');
console.log(buf.toString());

2. Streams

Streams are ideal for reading and writing data progressively. They help reduce memory usage and let your app respond sooner.

Readable

Source streams like files, HTTP requests, or sockets.

Writable

Destination streams like file writes or HTTP responses.

Duplex

Both read and write, such as TCP sockets.

3. Pipe example

const fs = require('fs');
const { createReadStream, createWriteStream } = require('fs');

const readStream = createReadStream('input.txt');
const writeStream = createWriteStream('output.txt');

readStream.pipe(writeStream);
