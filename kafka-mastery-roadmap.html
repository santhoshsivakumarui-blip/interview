offset 001 — core concepts

The building blocks

Get these fluent before touching a microservice. Everything downstream is a combination of these primitives.

Topics & partitions

A topic is a named log split into ordered partitions. Order is guaranteed within a partition only — never across the whole topic.

Offsets

Each message's position in a partition. Consumers track offsets to know what they've already processed.

Replication & ISR

Each partition is copied to follower brokers. The in-sync replica set determines what's safely durable.

Producers & acks

acks=0/1/all trade latency for durability. Idempotent producers prevent duplicate writes on retry.

Consumer groups

Partitions are divided among group members. Add consumers to scale; rebalancing reassigns work.

Delivery semantics

At-most-once, at-least-once, exactly-once — pick deliberately per use case, not by default.

offset 002 — microservice patterns

Kafka as the nervous system

Outbox pattern

Write the DB change and the event in one local transaction, then relay it to Kafka — avoids the dual-write problem.

Choreography vs orchestration

Let services react independently to events, or have a saga coordinator drive multi-step workflows explicitly.

Schema Registry

Avro or Protobuf schemas with compatibility rules stop one service's change from breaking every consumer.
