Array & Object Operations Guide

A practical guide to filter, sort, paginate, search, and transform arrays and objects using simple and complex patterns.

1. Filter, sort, and search

Use built-in methods when the logic is straightforward. Use a more deliberate approach when the logic becomes conditional or multi-step.

Filter

Use Array.filter for simple inclusion rules like active users or matching categories.

Sort

Use Array.sort with a comparator for ordered results like created date or name.

Search

Use a debounced input handler and filter the list on the client side for moderate data sizes.

Pagination

Use slice with page and limit to show only a subset of results.

const items = [{ name: 'Ada', active: true, score: 10 }, { name: 'Bob', active: false, score: 8 }];

const filtered = items.filter(item => item.active);
const sorted = [...filtered].sort((a, b) => b.score - a.score);
const page = sorted.slice(0, 5);

2. Debounced search

For search input, debounce avoids firing the filtering logic on every keystroke.

function debounce(fn, delay = 300) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

const onSearch = debounce((value) => {
  const result = items.filter(item => item.name.toLowerCase().includes(value.toLowerCase()));
  render(result);
}, 300);

3. Simple vs complex operations

Simple

Use filter, map, reduce, some, every, find, includes, slice, sort.

Complex

Use custom reducers, memoization, grouping logic, or pipelines when rules depend on multiple conditions.

Objects

Use Object.keys, Object.values, Object.entries, Object.fromEntries, and structured transforms.

Maps / Sets

Use Map and Set when you need unique keys, faster lookups, or custom grouping.

4. Object transformations

const users = { alice: 30, bob: 25 };

const entries = Object.entries(users);
const sortedEntries = entries.sort((a, b) => a[1] - b[1]);
const result = Object.fromEntries(sortedEntries);

const values = Object.values(users);
const keys = Object.keys(users);

5. When to choose which approach

Small dataset

Simple array methods are usually enough and easier to read.

Large or repeated dataset

Consider memoization, indexing, or server-side pagination and search.

Nested objects

Use recursive helpers or libraries for deep traversal and updates.

Performance-sensitive flows

Profile first; avoid premature optimization but use Map, Set, and memoization where needed.
