Service Workers in Detail

Service workers sit between the browser and the network, enabling offline support, caching, and smooth app experiences for animations and repeated interactions.

1. What a service worker is

A service worker is a special script that runs in the background, separate from the page, and can intercept network requests, cache assets, and control how the app behaves offline.

Background script

Runs independently of the page and survives refreshes and navigation.

Network proxy

Can intercept fetch requests and decide whether to serve cached content or go to the network.

Offline support

Enables cached shells and assets so the app still loads even when connectivity is weak.

Push and sync

Supports background sync and push notifications for richer experiences.

2. Lifecycle

Register

The browser downloads and installs the service worker script.

Install

Cache shell files and static assets during the install phase.

Activate

The worker becomes active and controls pages in its scope.

Fetch

It can intercept requests and decide how to respond.

if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/sw.js').then(() => {
      console.log('Service worker registered');
    });
  });
}

3. Caching strategy for animation-heavy apps

For animation or interactive experiences, caching should focus on stable assets like the app shell, JavaScript bundles, CSS, and icons, while avoiding stale or overly broad caching rules.

Cache-first for static assets

Great for fonts, icons, and bundled JS that rarely change.

Network-first for HTML

Useful for pages that need the latest content but can fall back to cache.

Stale-while-revalidate

Serves cached content quickly while updating it in the background.

self.addEventListener('fetch', (event) => {
  if (event.request.destination === 'document') {
    event.respondWith(fetch(event.request).catch(() => caches.match('/index.html')));
    return;
  }

  event.respondWith(
    caches.match(event.request).then((cached) => cached || fetch(event.request))
  );
});

4. Why service workers matter in animation

Animation apps often need fast load times, minimal stutters, and stable assets. A service worker helps by serving cached resources instantly and keeping the experience available even when the network is slow.

The app shell loads quickly, so the user sees the interface immediately.
Animation assets and scripts are reused from cache instead of re-downloading repeatedly.
Offline or flaky networks no longer break the experience completely.
Background sync can later send user actions that were queued while offline.

5. Interview takeaway

A strong interview answer is: service workers are background scripts that improve performance and resilience by intercepting requests, caching assets, and enabling offline behavior without blocking the main thread.
