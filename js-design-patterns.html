creational — singleton

One instance, no matter how many times you ask

Every call to get the instance returns the exact same object — useful for a config store, a connection pool, or a single logger.

new Config()
new Config()
new Config()
one shared instance
class Config {
  static #instance;
  constructor() {
    if (Config.#instance) {
      return Config.#instance;
    }
    Config.#instance = this;
  }
}
// every `new Config()` returns the same object

structural — module (revealing module)

Private state stays private. Only what's deliberately returned is reachable from outside.

const account = (function() {

structural — proxy

A gatekeeper in front of the real object

Same interface as the real thing, but it can cache, check access, or lazily create the object before ever touching it.

client
CachingProxy
RealAPIonly hit on a miss
function createCachingProxy(realApi) {
  const cache = new Map();
  return {
    get(id) {
      if (cache.has(id)) return cache.get(id);
      const result = realApi.get(id);
      cache.set(id, result);
      return result;
    }
  };
}

structural — composite

Treat one item and a group of items the same way

A folder and a file both respond to the same call — getSize() — so trees of either can be handled uniformly.

class Folder {
  children = [];
  getSize() {
    return this.children.reduce((sum, c) => sum + c.getSize(), 0);
  }
}

class EventBus {
  #listeners = {};
  on(event, fn) { (this.#listeners[event] ??= []).push(fn); }
  emit(event, data) {
    (this.#listeners[event] || []).forEach(fn => fn(data));
  }
}

behavioral — command

Turn an action into an object you can queue, log, or undo

Instead of calling a function directly, wrap the request as an object — now it can be queued, logged, retried, or reversed.

class AddTextCommand {
  constructor(doc, text) { this.doc = doc; this.text = text; }
  execute() { this.doc.push(this.text); }
  undo()    { this.doc.pop(); }
}

const history = [];
function run(cmd) { cmd.execute(); history.push(cmd); }
function undoLast() { history.pop()?.undo(); }
behavioral — chain of responsibility

Pass the request along until someone handles it

Each handler decides: handle it, or pass it on. The sender doesn't know or care which one eventually does.

class Approver {
  constructor(limit, next) { this.limit = limit; this.next = next; }
  handle(amount) {
    if (amount <= this.limit) return "approved";
    return this.next?.handle(amount) ?? "rejected";
  }
}

reference — when to reach for each

Quick recall

Singleton

Exactly one shared instance needed — config, connection pool, logger.

Factory Method

Object creation logic is complex or type varies at runtime.

Abstract Factory

You need a whole family of related objects guaranteed to match.

Builder

Constructing an object needs many optional steps or configurations.

Prototype

Cloning an existing object is cheaper than building one from scratch.

Decorator

Add behavior (logging, caching, retries) without touching the original function.

Module

Keep internal state private, expose only a deliberate public API.

Facade

Hide a messy set of subsystems behind one simple, stable entry point.

Adapter

Make an incompatible or legacy interface fit the shape your code expects.

Proxy

Add caching, access control, or lazy loading in front of a real object.

Bridge

Two independent hierarchies need to combine without multiplying classes.

Composite

Individual items and groups of items need to be treated identically.

Flyweight

Huge numbers of objects share most of their state — share it, don't copy it.

Observer

One event, several independent reactions — the backbone of event-driven UIs and services.

Strategy

Same operation, several interchangeable algorithms chosen at runtime.

Command

Actions need to be queued, logged, retried, or undone.

Chain of Responsibility

A request should pass through handlers until one can deal with it.

Iterator

Traverse a collection without exposing how it's structured internally.

Mediator

Components would otherwise need direct references to every other one.

Memento

Undo/redo or history needs a snapshot without breaking encapsulation.

State

Behavior depends heavily on current state, replacing sprawling if/else.

Template Method

An algorithm's shape is fixed, but one or two steps vary by subclass.

Visitor

New operations are needed across a class hierarchy you can't keep editing.
