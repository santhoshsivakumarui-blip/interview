Web Workers in Animation

Animation feels smooth when the main thread focuses on rendering, while heavy computation runs in a background worker.

1. Why workers matter for animation

The browser main thread is responsible for JavaScript execution, DOM updates, layout, and paint. If a long-running task blocks that thread, the animation frame gets delayed and the UI feels janky.

Main thread

Handles rendering, event listeners, DOM mutations, and user interaction.

Worker thread

Runs CPU-heavy work in parallel without blocking the UI thread.

Animation loop

Uses requestAnimationFrame to update visuals at a smooth, browser-friendly pace.

Best fit

Ideal for physics, pathfinding, simulation, large data transforms, and background calculations.

2. Typical architecture

UI thread

Keep it focused on input, UI state, and drawing. Use requestAnimationFrame for frame updates.

Worker thread

Compute positions, simulate motion, or process data and send results back when ready.

Message passing

Use postMessage for requests and onmessage for responses between the worker and the page.

const worker = new Worker('animation-worker.js');

worker.postMessage({ type: 'start', fps: 60 });

worker.onmessage = (event) => {
  const { positions } = event.data;
  renderFrame(positions);
};

3. Worker example

This worker computes a set of animation positions in the background and sends them back to the page for drawing.

// animation-worker.js
self.onmessage = (event) => {
  const { count } = event.data;
  const positions = [];

  for (let i = 0; i < count; i += 1) {
    positions.push({
      x: Math.sin(i / 10) * 100 + 200,
      y: Math.cos(i / 12) * 80 + 200,
    });
  }

  self.postMessage({ positions });
};

4. Best practices

Use requestAnimationFrame on the main thread for visual updates, not heavy computation.
Do not mutate the DOM from the worker; send computed data back and render it on the main thread.
Keep worker messages small and batch updates to reduce serialization overhead.
Use workers for expensive tasks, but avoid them for simple UI state changes.
Be careful with shared memory and browser support when deciding between workers and other approaches.

5. Interview takeaway

A strong answer is: Web Workers help animations stay smooth because they move heavy computation off the main thread, while the main thread remains free to handle rendering and interaction.
