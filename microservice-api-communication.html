offset 000 — the fundamental choice

Sync API call, or async event?

This decision shapes your whole architecture. Toggle to compare what each looks like when Order Service needs Inventory Service to do something.

Choose sync when

You need an immediate answer to act on — checking stock before showing "add to cart", validating a payment method.

Choose async when

The caller doesn't need to wait — sending a confirmation email, updating analytics, triggering a background job.

Most systems need both

APIs for read-your-writes UX; events for everything that can happen eventually and reliably.

offset 001 — core patterns

What sits between your services

The infrastructure that turns "services calling each other" into a system that doesn't fall over.

API Gateway

Single entry point: auth, rate limiting, routing, request aggregation, and TLS termination in one place.

Service discovery

Services register themselves (Consul, Eureka, or k8s DNS) so callers never hardcode IPs or ports.

Load balancing

Client-side or server-side distribution of requests across healthy instances of a service.

Circuit breaker

Stop calling a failing dependency for a cooldown period instead of piling up timeouts.

Retry with backoff

Transient failures get retried with exponential backoff and jitter — never a tight retry loop.

Timeouts & bulkheads

Every call has a deadline; isolate resource pools so one slow dependency can't starve everything else.

offset 003 — protocol choice

REST, gRPC, GraphQL, WebSocket

Pick the wire format based on the traffic pattern, not familiarity.

Request / response

REST over HTTP/JSON

Simple, cacheable, human-debuggable. Default choice for public and most internal APIs.

High throughput, internal

gRPC

Binary protobuf over HTTP/2, strict contracts, bidirectional streaming. Great for service-to-service calls inside the mesh.

Flexible reads

GraphQL

Clients ask for exactly the fields they need — useful at an aggregation layer in front of several services.

Live updates

WebSocket / SSE

Persistent connection for server-pushed updates — live order status, chat, dashboards.
