Child Process in Node.js

A child process is a separate OS process that your Node program can launch to run commands, scripts, or tools.

1. Why child processes exist

Sometimes you need to run a shell command, a Python script, or a system tool from Node. Child processes let you do that while keeping your main app process separate.

Isolation

Each child process has its own memory and execution context.

System tasks

Run ffmpeg, git, python, npm, or other binaries from Node.

IPC

Communicate with the parent over stdin, stdout, stderr, and stdio pipes.

2. Example

const { exec } = require('child_process');

exec('node --version', (error, stdout, stderr) => {
  if (error) console.error(error);
  else console.log(stdout.trim());
});
