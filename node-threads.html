Threads in Node.js

Node.js is single-threaded by default for JavaScript execution, but it can use threads for CPU-heavy work through worker threads and the libuv event loop.

1. What the main thread does

The main thread runs your JavaScript, handles requests, and coordinates events. It is great at I/O, but CPU-heavy work can block it.

Main thread

Runs your application code and event loop.

Event loop

Schedules callbacks and handles asynchronous work.

I/O

Async file system and network operations are offloaded to libuv.

2. Worker threads

Use Worker Threads when you need real parallel CPU work. They run in separate JavaScript execution contexts.

const { Worker } = require('worker_threads');
const worker = new Worker(`
  const { parentPort } = require('worker_threads');
  parentPort.postMessage('ready');
`);

worker.on('message', (msg) => console.log(msg));

3. When to use threads

Use worker threads for CPU-intensive tasks like image processing, encryption, parsing large data, or compression.
Do not use threads for simple network or file I/O, because Node already handles that efficiently with async APIs.
Threads introduce coordination cost and more complexity, so use them only when the benefit is real.
