ECS / EKS / Fargate

Run containers instead of managing OS-level instances yourself. Fargate removes even the worker nodes.

Lambda

Event-driven, no server to manage, billed per invocation — ideal for the async work Kafka consumers often do.

Auto Scaling Groups

Add or remove EC2 instances based on CPU, request count, or a schedule.

Elastic Load Balancer

ALB for HTTP(S) routing, NLB for raw TCP/UDP at very high throughput.

Route 53

DNS with health checks and routing policies — latency-based, weighted, or failover.

CloudFront

CDN that caches static assets at edge locations, cutting latency and origin load.

ACM

Free, auto-renewing TLS certificates attached directly to your load balancer.

NAT Gateway

Lets private-subnet instances reach the internet outbound without being reachable inbound.

offset 002 — storage & databases

Where the data actually lives

S3

Object storage for files, backups, and static hosting. Versioning and lifecycle rules manage cost over time.

RDS

Managed relational databases with Multi-AZ failover, automated backups, and read replicas.

DynamoDB

Managed NoSQL with single-digit millisecond latency at any scale, no servers to patch.

EFS

Shared file storage mounted across multiple instances or containers at once.

offset 003 — CI/CD & infrastructure as code

Ship without clicking in the console

If a step isn't in version control, it isn't repeatable — and it definitely isn't auditable.

CodePipeline / CodeBuild / CodeDeploy

Native AWS CI/CD, or wire the same stages through GitHub Actions or Jenkins.

ECR

Private container registry that CodeBuild pushes to and ECS/EKS pulls from.

CloudFormation

AWS-native infrastructure as code — declarative templates, drift detection, stack rollback.

Terraform / CDK

Multi-cloud or code-first infrastructure definitions, often preferred for larger teams.

offset 005 — observability

Knowing what's actually happening

CloudWatch

Metrics, logs, and alarms across every service — the default place an on-call engineer looks first.

CloudWatch Dashboards

One view combining ECS CPU, RDS connections, and ALB latency during an incident.

X-Ray

Distributed tracing across services — the same idea as the trace ID from the API communication page, AWS-native.

Alarms → SNS
