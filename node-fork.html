Fork in Node.js

Fork creates a new Node process that is a copy of the current one, often used for worker-like behavior and IPC.

1. What fork means

A forked process starts as a child process that inherits the parent’s memory state at the moment of forking, then continues independently.

Parent process

Creates and controls the child worker.

Child process

Runs its own copy of the application and can communicate over IPC.

IPC

Use process.send and on('message') for communication.

2. Example

const { fork } = require('child_process');
const child = fork('./worker.js');

child.on('message', (msg) => console.log('Parent received:', msg));
child.send({ task: 'start' });
